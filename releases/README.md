# Releases

This folder contains versioned release notes for the project.

## How the release system works

Each release is documented as an optional update milestone:

- Update 1.0 is the first public milestone
- Update 1.1 is the next optional follow-up release
- Future updates will continue in the same pattern

## Release philosophy
- Releases are open-source and optional
- Users are never forced to upgrade
- Older versions remain usable and can be kept indefinitely
- Updates are documented as milestones, not mandatory changes
- Users can roll back to an older update if they prefer
