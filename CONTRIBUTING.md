# Contributing to Palladium

Thank you for contributing! Palladium is a community-driven project and we welcome all contributions.

## Ways to Contribute

- **Code**: Bug fixes, new features, improvements
- **Documentation**: Guides, API docs, tutorials
- **Marketplace Tools**: Submit new `.tool` files
- **Plugins**: Create and share plugins
- **Testing**: Report bugs, test releases
- **Translations**: Help localize the UI

## Development Setup

### Prerequisites

- Bash 4+ (Linux/macOS/WSL2/Git Bash)
- Docker & Docker Compose
- ShellCheck (`apt install shellcheck` or `brew install shellcheck`)
- Python 3.11+ (for API server)

### Clone & Setup

```bash
git clone https://github.com/M-2000-0/Palladium.git
cd Palladium

# Run tests
./tests/run.sh

# Lint
shellcheck palladium/modules/*.sh
```

## Code Style

### Bash Guidelines

- Use `#!/bin/bash` shebang
- Enable `set -euo pipefail` in scripts (not in interactive modules)
- Use descriptive variable names: `service_name` not `sn`
- Quote all variables: `"$var"` not `$var`
- Use `local` for function variables
- Functions: `snake_case`
- Constants: `UPPER_SNAKE_CASE`
- Max line length: 100 chars

### ShellCheck

All code must pass ShellCheck:

```bash
# Run on all modules
shellcheck --shell=bash --source-path=palladium/modules palladium/modules/*.sh

# Ignore specific warnings if needed
# shellcheck disable=SC2034,SC2154
```

### Git Commits

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add Ollama model manager
fix: resolve sed -i macOS compatibility
docs: update API reference
chore: update dependencies
refactor: simplify backup logic
test: add unit tests for security module
```

## Pull Request Process

1. **Fork** the repository
2. **Create branch**: `git checkout -b feat/amazing-feature`
3. **Make changes** with tests
4. **Run checks**: `./tests/run.sh && shellcheck ...`
5. **Commit** with conventional message
6. **Push** and open PR

### PR Requirements

- [ ] All tests pass
- [ ] ShellCheck passes
- [ ] Documentation updated
- [ ] CHANGELOG.md updated (under `[Unreleased]`)
- [ ] No merge conflicts

## Adding Marketplace Tools

### Tool File Format (`.tool`)

```yaml
# Generated by marketplace_submit
name: tool-name
desc: Short description
image: docker/image:tag
port: 8080
category: ai|data|automation|web|devops|messaging|storage
volumes: ./data:/data
env: KEY=value,KEY2=value2
```

### Submission Steps

1. Run `palladium marketplace` → Custom Install → Submit Tool
2. Or manually create `.tool` file in `palladium/marketplace/`
3. Test: `palladium marketplace install tool-name`
4. Open PR with the new file

### Categories

| Category | Description |
|----------|-------------|
| `ai` | AI/ML tools, LLMs, vector DBs |
| `data` | Databases, analytics, BI |
| `automation` | Workflows, scheduling, CI/CD |
| `web` | Proxies, web servers, apps |
| `devops` | Monitoring, logging, containers |
| `messaging` | Message brokers, MQTT, queues |
| `storage` | Object storage, file sync |

## Adding Plugins

### Plugin Structure

```
plugins/available/my-plugin/
├── my-plugin.sh      # Main entry point
├── plugin.yml        # Metadata
└── README.md         # Documentation
```

### Plugin Metadata (`plugin.yml`)

```yaml
name: my-plugin
version: 1.0.0
description: What this plugin does
author: Your Name
license: MIT
entry: my-plugin.sh
commands:
  - cmd: my-command
    desc: Description
hooks:
  - menu_main
```

### Plugin Entry Point

```bash
#!/bin/bash
# Plugin: my-plugin

plugin_my-plugin_init() {
    log_message "Plugin my-plugin initialized" "DEBUG"
}

plugin_my-plugin_menu() {
    echo "  [x] My Plugin - Custom action"
}

plugin_my-plugin_handle() {
    case "$1" in
        x) echo "Custom action executed" ;;
    esac
}
```

## Testing

### Unit Tests

```bash
# Run all tests
./tests/run.sh

# Run specific test
bash tests/unit_test.sh
```

### Integration Tests

```bash
# Test Docker Compose configs
cd palladium/services
docker-compose -f n8n.yml config --quiet
docker-compose -f postgres.yml config --quiet
```

### Manual Testing Checklist

- [ ] Fresh install on clean system
- [ ] USB/SSD plug-and-play
- [ ] Service install/start/stop/remove
- [ ] Stack installation
- [ ] Marketplace install
- [ ] AI toolkit (Ollama, connectors)
- [ ] Backup/restore
- [ ] Security features (vault, 2FA)
- [ ] API server start/stop
- [ ] Plugin load/enable/disable

## Release Process

Maintainers only:

1. Update `VERSION` file
2. Update `CHANGELOG.md`
3. Create tag: `git tag v1.2.0`
4. Push tag: `git push origin v1.2.0`
5. GitHub Actions creates release
