# Project Roadmap

## 12-Month Plan: One Update Per Month

This roadmap is focused on the broader public goal: make the project feel like a flexible, easy-to-use portable server platform for many different users, not just one specific setup.

### Month 1 — August 2026: Plug-and-Play SSD Launch
- Make the SSD startup flow feel effortless and elegant
- Add a polished single-click launcher for Chromebook/Linux
- Improve first-run behavior for n8n, PostgreSQL, and starter stacks
- Goal: plug in the SSD, launch once, and get a complete local stack running smoothly

### Month 2 — September 2026: One-Click Starter Stack
- Deliver a premium starter experience with guided setup
- Make n8n + PostgreSQL install and launch feel seamless and reliable
- Add polished status messaging, launch URLs, and onboarding feedback
- Goal: first-time users should feel the product is professional and easy to trust

### Month 3 — October 2026: Premium Terminal Dashboard
- Improve the terminal dashboard with modern visuals, clear state indicators, and richer interaction
- Show running services, health, ports, and quick actions in one place
- Goal: users should understand the system at a glance inside the terminal, without needing a separate UI

### Month 4 — November 2026: Backup, Restore, and Migration
- Make backup and restore flows robust, safe, and stress-free
- Improve portability between SSDs, laptops, and different hosts
- Goal: users can move their environment between machines with confidence

### Month 5 — December 2026: Security by Default
- Add stronger defaults for local-only hosting and home use
- Improve secret handling, local access controls, and basic protection guidance
- Goal: the system should feel secure and production-ready without being intimidating

### Month 6 — January 2027: App Templates and Presets
- Add high-quality presets for automation, local AI, web apps, and data workflows
- Make it easy to launch common setups with one command or one click
- Goal: users can start from a polished template instead of building everything manually

### Month 7 — February 2027: Chromebook and Linux Excellence
- Improve ChromeOS and Linux compatibility to a first-class standard
- Reduce manual setup steps to near-zero for supported environments
- Goal: Chromebook users should feel the experience is truly native and reliable

### Month 8 — March 2027: Remote Access and Networking
- Add optional secure remote access patterns for trusted environments
- Improve local networking, reverse proxy support, and access guidance
- Goal: the platform should support both local-first and flexible remote use cases

### Month 9 — April 2027: Monitoring, Alerts, and Health
- Add clearer monitoring for CPU, memory, disk, and service health
- Introduce lightweight alerts and recovery guidance for failed services
- Goal: users should be able to maintain a healthy setup with minimal effort

### Month 10 — May 2027: Marketplace Expansion
- Expand the marketplace with high-value self-hosted tools and app bundles
- Improve discoverability, metadata, and one-click install quality
- Goal: the platform should feel like a serious, feature-rich ecosystem

### Month 11 — June 2027: Migration and Clone Workflow
- Make cloning from one drive to another polished and reliable
- Add clear guided workflows for moving to new hardware or a new host
- Goal: migrating a complete setup should feel easy and professional

### Month 12 — July 2027: v2.0 Beta Release
- Bundle the major improvements into a polished beta release
- Focus on reliability, documentation, polish, and developer experience
- Goal: the project should feel like a best-in-class open-source portable server platform

## Recommended focus for the first 3 months
1. Make the SSD/Chromebook setup feel seamless
2. Make n8n and the starter stack effortless
3. Make the app feel understandable from the first launch
